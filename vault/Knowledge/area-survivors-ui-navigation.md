# AreaSurvivors UIフォーカス/パッド操作メモ

## 基本方針

- 全画面で、上下左右キーは見た目上近いSelectableへ直感的に移動する。
- 画面外や非表示のSelectableへフォーカスを飛ばさない。下端/上端では現在のフォーカスを維持する。
- マウス操作とパッド操作は入力モードとして分ける。パッド操作時はフォーカス追従、マウスホバー/スクロール時はマウス操作を優先する。
- 毎フレームでフォーカス位置へスクロールを戻さない。フォーカス追従は「パッドの上下左右で選択が変わった時」など操作起因で行う。

## UI実装の注意

- ユーザーがScene上で配置したボタン、パネル、スクロール範囲をRuntimeで固定値へ戻さない。
- キーコンフィグはテキスト入力欄ではなく、設定ボタン押下後の「入力待ち」状態で次のキー/ボタンを記録する方式が分かりやすい。
- PS系の表記はデフォルトで×を決定、〇をキャンセルにする。Unity/InputManager側のボタン番号は環境でずれるため、実機確認とラベル表示をセットで行う。
- SliderなどSelectableだが見た目上フォーカスが分かりにくいUIは、選択時の背景色/アウトラインを別途出す。左右入力で値変更できても、上下で次項目へ移動できる状態を維持する。
- リスト/図鑑/オプションなどスクロール画面では、マウスホイールやドラッグスクロールをフォーカス追従より優先する。

## 強化画面固有

- 強化画面はScrollRectではなくズーム/パンのため、一般的なスクロール追従ではフォーカス位置を画面内へ収められない。選択変更時にスキルツリーViewport側のパンを調整する。
- マウス操作中はホバー中ノードのツールチップだけを出し、ホバーが外れたらツールチップを消す。パッド操作中だけフォーカス中ノードのツールチップを出す。
